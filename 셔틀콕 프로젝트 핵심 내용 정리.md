지금까지 논의하고 결정한 **'셔틀콕'** 프로젝트의 핵심 내용들을 깔끔하게 총정리해 줄게\! 🚀

### ---

**\#\# 📌 1\. 프로젝트 개요 및 기술 스택**

* **서비스명:** 셔틀콕  
* **핵심 컨셉:** 누구나 정보를 올리고, 수정할 수 있는 나무위키 컨셉의 참여형 대기업 셔틀 지도 웹앱.  
* **프론트엔드:** React (Vite 기반) \+ TypeScript.  
* **백엔드:** Firebase (Firestore, Auth, Cloud Functions) 기반의 서버리스 아키텍처.  
* **지도 API:** 네이버 Web Dynamic Map API (지도 상호작용 및 마커 클러스터링을 위해 필수 선택).  
* **플랫폼:** PWA (Progressive Web App)를 적용하여 크롬 등에서 네이티브 앱처럼 설치 기능 지원.

### **\#\# ✨ 2\. 핵심 기능 리스트**

* **지도 기반 위키:** 로그인한 누구나 특정 위치에 핀을 꽂고 셔틀 정보(기업명, 출퇴근, 시간, 혼잡도)를 저장 및 조회. 한 위치에 여러 셔틀 정보 중복 저장 가능.  
* **정보 수정 및 혼잡도 관리:** 잘못된 위치나 시간은 '수정 요청'을 받고, 혼잡도(여유/적당/부족)는 누구나 즉시 변경 가능.  
* **밀집도 시각화:** 출/퇴근 토글에 맞춰 셔틀이 많이 다니는 지역을 마커 클러스터링으로 묶고, 대수에 따라 색깔 동그라미(빨강, 노랑, 회색 등)와 숫자로 강조 표시.  
* **게이미피케이션:** 정보 기여자에게 포인트를 부여하고 누적 및 일일 Top 3 랭킹 제공.  
* **사용자 인증:** 구글 로그인(Firebase Auth) 기반이며, 닉네임은 중복되지 않도록 처리.

### **\#\# 🗄️ 3\. 데이터베이스 모델링 (Firestore)**

* **users:** 유저 정보 (uid, email, nickname, company, points 등).  
* **nicknames:** 닉네임 중복 검사용 컬렉션.  
* **stations:** 지도에 찍히는 핀(위/경도) 정보.  
  * **shuttles (서브 컬렉션):** 해당 핀에 정차하는 구체적인 셔틀 정보들.  
* **correction\_requests:** 위치/시간 수정 요청 내역이 쌓이는 컬렉션.

### **\#\# 📁 4\. React 프로젝트 폴더 구조 (src/ 내부)**

앞으로의 개발을 탄탄하게 받쳐줄 확장성 높은 구조야.

* **components/**: 지도 위에 띄울 핀(MapMarker), 출퇴근 전환 토글(Toggle) 등 재사용 가능한 UI 조각들.  
* **pages/**: 메인 지도 화면(MapPage), 로그인 화면(LoginPage) 등 큼직한 전체 화면 단위 컴포넌트.  
* **hooks/**: 네이버 지도 컨트롤(useNaverMap)이나 유저 상태 관리(useAuth) 등 상태 관리 및 비즈니스 로직.  
* **services/**: firebase.ts 등 데이터베이스 연동 및 외부 API 통신 로직.  
* **types/**: TypeScript 타입 및 인터페이스 정의 파일들.  
* **utils/**: 시간 변환, 포인트 계산 등 화면에 종속되지 않은 공통 헬퍼 함수들.
